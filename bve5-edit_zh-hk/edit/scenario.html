檔案格式

BveTs Scenario 2.00

檔案格式必須要記述於場景檔案的開端。


文字編碼

:encoding

在檔案格式以後一行，請先以「:」，然後再輸入文字編碼名稱，然後再記述其他內容。請勿於文字編碼名稱前方或後方輸入空白字符。如果不對檔案文本的編碼作出設定，程式會以 UTF-8 讀取餘下內容。

以下為使用 Shift-JIS 作為編碼的例子：

BveTs Map 2.00:shift_jis


註釋

#comment

該行文字如以 # （井號）或 ; （分號）作為開端的話將會被無視。


引用路線檔案

Route = filePath1 * weight1 | filePath2 * weight2 | ...

可以使用相對路徑記述路線檔案的位置。在記述多個路線檔時請以 "|" 區隔開每個檔案路徑，開啟場景檔案時會隨機選用其中一個路線檔案。要設定每個檔案的機會率，請於檔案路徑後方記述。機會率值越大，該路線檔被隨機選用的機會則越大。如果將機會率值省略的話，該路線檔案的機會率則為 1。

例如：
Route = mackoy.Keisei\Map-Dry.txt * 2.5 | mackoy.Keisei\Map-Wet.txt


引用列車檔案

Vehicle = filePath1 * weight1 | filePath2 * weight2 | ...

可以使用相對路徑記述列車檔案的位置。記述方法如同引用路線檔案。
