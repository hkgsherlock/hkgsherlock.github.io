Vehicle Performance Table File

記述格式

以 CSV 格式（逗號分隔型取值格式）作為基本記述格式。

BveTs Vehicle Performance Table 1.00 此首行文字必須記載檔案格式。
#comment
 以「#」（分號）作為該行文字的開端的話將會被無視。
speed1,step1value1,step2value1,...
speed2,step1value2,step2value2,...
...

拉力表、電流表及階段進行屬性表中的數值列數必須相同。

拉力表

在第一欄中，請由低至高速記述行駛速度[公里/小時]。值與值之間不必擁有相同間距
在第二欄以後，請記述該行駛速度時每一卡列車所對應的拉力[N]。每一欄所列出的值都代表一個特性曲線。而兩個值之間都會以線連着。

電流表

在第二欄以後，請記述該行駛速度時主電機電流[A]。此與拉力表相同。

階段進行屬性表

定義特性曲線間的切換條件。原本於第一欄中記述的行駛速度請改為記術以下項目。在第二欄以後，則請記述各項目所對應的特性曲線。這在此叫做一個特性曲線格數。

名稱 單位 說明
BreakerDelay_On s 斷路器的延遲時間（通電）。
BreakerDelay_Off s 斷路器的延遲時間（斷電）。
CurrentReducingTime s 電流阻斷時電流下降的時間。如果不想減少電流阻斷的話請設為 0。
JerkRegulation_Up A/s 搖晃控制（増加）。如果要停用搖晃控制的話請設為 0。
JerkRegulation_Down A/s 搖晃控制（減少）。如果要停用搖晃控制的話請設為 0。
RequiredNotch_Up   加速或抑速時，轉至下一個階段所必要的最低控制桿格數。此與常用制動時並無關係。
RequiredNotch_Down   加速或抑速時，轉至上一個階段所必要的最高控制桿格數。如果它不返回到以前的階段的話請設此值為 0 。如果它不返回到惰行時的初始階段則請設此值為 -1 。
StepDelay_Up s 升一格所需的時間。
StepDelay_Down s 降一格所需的時間。
CurrentLimitingValue_Empty A 可進行到下一個階段的最大電流（空車時）。
CurrentLimitingValue_Full A 可進行到下一個階段的最大電流（負荷最大時）。
